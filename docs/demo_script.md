# Demo Script — DataPulse Commerce

## 60-120 Second Portfolio Video

### 0-10s — Opening

DataPulse Commerce is a full-stack e-commerce portfolio app built with Next.js, FastAPI, PostgreSQL, and a mock payment flow.

It demonstrates a complete commerce path from storefront activity to admin operations and analytics.

### 10-30s — Storefront And Product Flow

Start on the storefront and open the product catalog.

Show the product listing, then open a product detail page. Point out that the catalog is backed by real API records, product variants, stock availability, and inventory-aware cart behavior.

Add a product to the cart.

### 30-50s — Cart And Checkout

Open the cart and show the order summary.

Sign in as the customer demo user, then open checkout. The checkout is authenticated and uses a safe mock payment provider, so no real card data is collected.

Show the contact, shipping, billing, payment, and order summary sections.

### 50-75s — Customer Account

Open the customer account area.

Explain that customers can keep a persistent cart and review their order history. The demo data is synthetic and safe for public portfolio review.

### 75-100s — Admin Back Office

Sign in as the admin demo user.

Open the admin overview and briefly show orders, products, and inventory. Explain that the back office connects storefront behavior to operational workflows like order status, payment state, stock, shipments, and refunds.

### 100-120s — Analytics And Close

Open the analytics dashboard.

Explain that commerce activity feeds operational metrics and the existing DataPulse BI foundation, including revenue, conversion, payment health, and inventory risk.

Close with:

> DataPulse Commerce is deployed on Vercel, Render, and Supabase, with documented architecture, tests, screenshots, and a public demo ready for review.
